My development career has grown organically from a deep fascination with how software works under the hood. Starting in the game modding community — where the only way to learn was to read source code, experiment, and break things — I built a foundation in systems programming and low-level C++ that continues to inform everything I do. Over time, that curiosity expanded into web development, open-source authorship, and a broader appreciation for the full software stack.

## Open Source & Game Modding

The **MTA:San Andreas** ecosystem was my first real engineering environment. Building mods for a multiplayer GTA engine means working with constraints: legacy codebases, undocumented internals, strict performance budgets, and a passionate but exacting community of players and developers.

### Midnight Purple SA

My most significant project in this space is **[Midnight Purple SA](/midnight-purple/introduction)** — a custom fork of MTA:San Andreas that replaces the default BASS audio engine with **FMOD Core SDK**. This was not a trivial integration; it required:

- Reverse-engineering the existing audio pipeline and identifying all hook points
- Writing a clean C++ abstraction layer between MTA:SA's game logic and the FMOD runtime
- Exposing a comprehensive **Lua API** so server-side scripters can control 3D audio, reverb zones, DSP effect chains, and vehicle sound profiles — without touching C++
- Designing the system to be hot-reloadable and developer-friendly, with clear error surfaces

The result is a fork that brings console-grade audio fidelity to a 2004 open-world game, fully scriptable by the community.

### MTA:SA Ecosystem Contributions

Beyond Midnight Purple SA, my time in the MTA:SA community involved:

- Writing and maintaining Lua game-mode scripts for multiplayer servers
- Building and publishing open-source resources for the MTA community
- Diagnosing and patching engine-level bugs in collaborative forks
- Mentoring newer modders on Lua scripting patterns and C++ interop concepts

## Systems Programming

Across both game modding and standalone tools, I've accumulated significant experience in systems-level C++ development.

### Audio Engine Integration

Integrating FMOD into MTA:SA's runtime required understanding:

- **FMOD Core SDK** architecture — channel groups, DSP graphs, 3D listener/emitter models, reverb zones
- **Memory management** in a long-running game process — avoiding leaks, handling FMOD's internal allocator, and coordinating with the host process's heap
- **Thread safety** between the game's main thread, FMOD's mixer thread, and the Lua scripting VM
- **DirectX integration** — coordinating audio state with the game's rendering and physics ticks

### Native Build & Tooling

- Visual Studio 2022 for Windows-targeted C++ development, including MSVC-specific pragmas, DLL export tables, and linker configuration
- Linux cross-compilation workflows for server-side components
- DirectX SDK integration for rendering-adjacent systems work

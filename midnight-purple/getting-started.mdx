This guide walks you through the essentials of the Midnight Purple SA FMOD audio system: loading sound assets, triggering playback, applying 3D world-space positioning, adjusting volume, and routing channels into the correct audio category. By the end you'll have a working pattern you can drop into any client-side MTA:SA resource script.

<Note>
  All `fmod*` functions return `false` on failure. Use `fmodGetLastError()` to retrieve the error code and a human-readable description whenever something doesn't behave as expected.
</Note>

## Playing Your First Sound

<Steps>
  <Step title="Load a sound asset">
    `fmodCreateSound` loads an audio file into memory and returns a sound handle. The second parameter enables 3D spatialization; pass `false` for a standard 2D sound. The third parameter enables looping.

    ```lua
    local soundId = fmodCreateSound("sounds/engine.wav", false, false)
    if not soundId then
      local errCode, errStr = fmodGetLastError()
      outputDebugString("[FMOD] Failed to create sound: " .. tostring(errCode) .. " - " .. tostring(errStr))
      return
    end
    ```
  </Step>

  <Step title="Play the sound (2D, non-spatialized)">
    `fmodPlaySound` starts playback and returns a channel handle you use for all subsequent control calls.

    ```lua
    local channelId = fmodPlaySound(soundId)
    if not channelId then
      local errCode, errStr = fmodGetLastError()
      outputDebugString("[FMOD] Playback failed: " .. tostring(errCode) .. " - " .. tostring(errStr))
      return
    end
    ```
  </Step>

  <Step title="Adjust volume">
    Pass the channel handle and a value between `0.0` (silent) and `1.0` (full volume) to `fmodSetSoundVolume`.

    ```lua
    fmodSetSoundVolume(channelId, 0.8)  -- 80% volume
    ```
  </Step>

  <Step title="Free the sound when done">
    Releasing the sound handle frees the audio data from memory. You only need to do this manually if you want to free the sound before the resource stops — per-resource tracking handles cleanup automatically on resource stop.

    ```lua
    fmodFreeSound(soundId)
    ```
  </Step>
</Steps>

<Tip>
  Assign channels to categories (`sfx`, `ambient`, `music`) to control volumes independently across all sounds in that category at once.
</Tip>

## Playing a 3D Spatial Sound

Pass `true` as the second argument to `fmodCreateSound` to enable 3D mode, then supply world-space coordinates and min/max audibility distances when calling `fmodPlaySound`. Positions use GTA:SA world axes directly — no remapping needed, and the audio listener tracks the local player's position automatically.

```lua
-- Load as 3D sound
local soundId = fmodCreateSound("sounds/explosion.wav", true, false)

-- Play at world position x=100, y=200, z=10
-- minDistance=5  (full volume within 5 units)
-- maxDistance=80 (inaudible beyond 80 units)
local channelId = fmodPlaySound(soundId, 100, 200, 10, 5, 80)
```

## Streaming Long Audio Files

For music tracks, ambient loops, or any audio file too large to hold entirely in RAM, use `fmodCreateStream` instead of `fmodCreateSound`. Streaming reads data from disk in small chunks, keeping memory usage low regardless of file length.

```lua
-- Use fmodCreateStream for music or long audio (lower RAM usage)
local musicId = fmodCreateStream("audio/ambient_track.ogg", false, true)  -- looped

local ch = fmodPlaySound(musicId)

-- Route to the Music category and set its volume
fmodSetChannelCategory(ch, "music")
fmodSetCategoryVolume("music", 0.6)
```

`fmodSetCategoryVolume` adjusts the volume for **all** channels currently assigned to that category, making it easy to implement player-facing audio sliders for music, SFX, and ambient independently.

## Resource Cleanup

Midnight Purple SA tracks every sound created within a resource against that resource's lifetime. When a resource stops — whether intentionally via `stopResource` or due to a script error — all FMOD sounds and channels belonging to it are automatically released. You do **not** need to add cleanup logic to `onClientResourceStop` unless you want to free large assets earlier (for example, swapping sound banks mid-session).

To free a sound manually before the resource stops:

```lua
fmodFreeSound(soundId)   -- releases sound data; any active channel using it stops
```

Freeing a sound that is currently playing will stop the channel immediately. Always stop or reassign channels before freeing sounds in performance-sensitive code paths.

These functions give you fine-grained control over individual sound channels — adjusting volume, pitch, 3D world position, Doppler velocity, loop state, and geometry-based occlusion. Each function targets a specific channel by its `channelId` integer handle returned when a sound is played.

---

## fmodSetSoundVolume

```lua
fmodSetSoundVolume(channelId, volume)
```

Set the volume of a channel. Range is `0.0` (silent) to `1.0` (full volume). Values are clamped to this range internally by FMOD.

<ParamField path="channelId" type="integer" required>
  The channel handle returned by a play function (e.g. `fmodPlaySound`).
</ParamField>

<ParamField path="volume" type="number" required>
  Volume scalar in the range `0.0` (silent) to `1.0` (full volume).
</ParamField>

<ResponseField name="result" type="bool">
  Returns `true` on success, `false` if the channel handle is invalid or the call failed.
</ResponseField>

**Example**

```lua
fmodSetSoundVolume(ch, 0.75)
```

## fmodGetSoundVolume

```lua
fmodGetSoundVolume(channelId)
```

Read the current volume of a channel.

<ParamField path="channelId" type="integer" required>
  The channel handle to query.
</ParamField>

<ResponseField name="volume" type="number">
  The current volume scalar (`0.0`–`1.0`). Returns `false` if the channel is invalid.
</ResponseField>

**Example**

```lua
local vol = fmodGetSoundVolume(ch)
outputChatBox("Current volume: " .. tostring(vol))
```

## fmodSetSoundPitch

```lua
fmodSetSoundPitch(channelId, pitch)
```

Set the pitch multiplier for a channel. `1.0` is normal pitch, `2.0` is one octave up, and `0.5` is one octave down. Useful for dynamic audio — engine RPM, slow-motion effects, or musical transposition.

<ParamField path="channelId" type="integer" required>
  The channel handle to modify.
</ParamField>

<ParamField path="pitch" type="number" required>
  Pitch multiplier. `1.0` = normal pitch. `2.0` = one octave up. `0.5` = one octave down.
</ParamField>

<ResponseField name="result" type="bool">
  Returns `true` on success, `false` if the channel handle is invalid.
</ResponseField>

**Example**

```lua
fmodSetSoundPitch(engineCh, 1.4)  -- engine running faster
```

## fmodSetSoundPosition

```lua
fmodSetSoundPosition(channelId, x, y, z)
```

Update the world-space position of a 3D channel. Call this every frame (or on a fast timer) to track a moving object. Has no effect on 2D (non-positional) channels.

<ParamField path="channelId" type="integer" required>
  The 3D channel handle to reposition.
</ParamField>

<ParamField path="x" type="number" required>
  World X coordinate.
</ParamField>

<ParamField path="y" type="number" required>
  World Y coordinate.
</ParamField>

<ParamField path="z" type="number" required>
  World Z coordinate.
</ParamField>

<ResponseField name="result" type="bool">
  Returns `true` on success, `false` if the channel is invalid or is a 2D channel.
</ResponseField>

**Example**

```lua
addEventHandler("onClientRender", root, function()
  local x, y, z = getElementPosition(myVehicle)
  fmodSetSoundPosition(engineCh, x, y, z)
end)
```

---

## fmodGetSoundPosition

```lua
fmodGetSoundPosition(channelId)
```

Read the current world position of a 3D channel. Returns `false` if the channel is 2D (non-positional) or invalid.

<ParamField path="channelId" type="integer" required>
  The channel handle to query.
</ParamField>

<ResponseField name="x, y, z" type="number">
  Three number values representing the current world-space position of the channel. Returns `false` if the channel is 2D or the handle is invalid.
</ResponseField>

**Example**

```lua
local x, y, z = fmodGetSoundPosition(engineCh)
if x then
  outputChatBox(("Sound at: %.1f, %.1f, %.1f"):format(x, y, z))
else
  outputChatBox("Channel is 2D or invalid.")
end
```

## fmodSetSoundVelocity

```lua
fmodSetSoundVelocity(channelId, vx, vy, vz)
```

Set the world-space velocity vector of a channel for Doppler pitch shift simulation. Higher velocity relative to the listener produces a Doppler effect — sounds approaching the listener pitch up, and sounds moving away pitch down. Scale the raw MTA velocity values to game units as needed.

<ParamField path="channelId" type="integer" required>
  The 3D channel handle to update.
</ParamField>

<ParamField path="vx" type="number" required>
  X component of the velocity vector in world space.
</ParamField>

<ParamField path="vy" type="number" required>
  Y component of the velocity vector in world space.
</ParamField>

<ParamField path="vz" type="number" required>
  Z component of the velocity vector in world space.
</ParamField>

<ResponseField name="result" type="bool">
  Returns `true` on success, `false` if the channel is invalid.
</ResponseField>

**Example**

```lua
local vx, vy, vz = getElementVelocity(myVehicle)
fmodSetSoundVelocity(engineCh, vx * 50, vy * 50, vz * 50)  -- scale to game units
```

## fmodSetSoundLooped

```lua
fmodSetSoundLooped(channelId, loop)
```

Change the loop mode on a live channel without restarting playback. Applies the FMOD `FMOD_LOOP_NORMAL` or `FMOD_LOOP_OFF` mode depending on the `loop` argument.

<ParamField path="channelId" type="integer" required>
  The channel handle to modify.
</ParamField>

<ParamField path="loop" type="bool" required>
  `true` to enable looping, `false` to disable. Looping takes effect at the next loop boundary.
</ParamField>

<ResponseField name="result" type="bool">
  Returns `true` on success, `false` if the channel handle is invalid.
</ResponseField>

**Example**

```lua
fmodSetSoundLooped(ambientCh, true)
```

---

## fmodGetSoundLooped

```lua
fmodGetSoundLooped(channelId)
```

Read the current loop state of a channel.

<ParamField path="channelId" type="integer" required>
  The channel handle to query.
</ParamField>

<ResponseField name="looping" type="bool">
  Returns `true` if the channel is currently set to loop, `false` if not. Returns `false` if the channel handle is invalid.
</ResponseField>

**Example**

```lua
if fmodGetSoundLooped(ambientCh) then
  outputChatBox("Ambient track is looping.")
end
```

## fmodSetChannelOcclusion

```lua
fmodSetChannelOcclusion(channelId, directOcclusion [, reverbOcclusion])
```

Apply geometry-aware muffling using FMOD's native `set3DOcclusion`. `directOcclusion` controls how much the direct signal is attenuated — `0.0` means no occlusion and `1.0` means the sound is fully blocked. FMOD applies a frequency-dependent low-pass filter internally so occluded sounds become muffled rather than simply quieter, producing a realistic through-wall effect. The optional `reverbOcclusion` parameter independently attenuates the reverb send level.

<ParamField path="channelId" type="integer" required>
  The channel handle to apply occlusion to.
</ParamField>

<ParamField path="directOcclusion" type="number" required>
  Direct signal attenuation in the range `0.0` (no occlusion) to `1.0` (fully blocked). FMOD applies a low-pass filter at higher values.
</ParamField>

<ParamField path="reverbOcclusion" type="number">
  Optional. Reverb send attenuation in the range `0.0` (no attenuation) to `1.0` (fully muted reverb). Defaults to `0.0` if not provided.
</ParamField>

<ResponseField name="result" type="bool">
  Returns `true` on success, `false` if the channel handle is invalid or the call failed.
</ResponseField>

**Example**

```lua
-- Muffle a sound behind a wall
fmodSetChannelOcclusion(engineCh, 0.7, 0.3)
-- Unmuffle
fmodSetChannelOcclusion(engineCh, 0.0)
```

## Update Loop Pattern

For moving sound sources, the idiomatic approach is to drive both position and velocity updates from a single timer or render handler. Using a `setTimer` with a short interval keeps overhead low while maintaining smooth 3D audio tracking:

```lua
local RENDER_INTERVAL = 50  -- ms
local timer = setTimer(function()
  if not isElement(myVehicle) then return end
  local x, y, z = getElementPosition(myVehicle)
  local vx, vy, vz = getElementVelocity(myVehicle)
  fmodSetSoundPosition(engineCh, x, y, z)
  fmodSetSoundVelocity(engineCh, vx * 50, vy * 50, vz * 50)
end, RENDER_INTERVAL, 0)
```

Stop the timer when the sound is no longer needed to avoid updating a destroyed channel:

```lua
if isTimer(timer) then killTimer(timer) end
fmodStopSound(engineCh)
```

<Tip>
  For vehicle audio, bind `fmodSetSoundPosition` to an `onClientRender` handler or a fast timer to keep 3D positions in sync with the game world. Missing position updates on fast-moving vehicles will cause audible spatial jumps and incorrect Doppler calculations.
</Tip>

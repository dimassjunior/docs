This page covers the four functions that manage the lifecycle of FMOD sound assets — loading, streaming, loading from memory, and freeing. All functions are available client-side and return `false` on failure; use `fmodGetLastError()` to retrieve the error code and description.

---

## fmodCreateSound

```lua
fmodCreateSound(path, [is3D=false], [loop=false])
```

Load a sound asset from a resource-relative file path. Best suited for short sounds such as SFX or one-shots that can be kept entirely in memory. For long audio files, prefer [`fmodCreateStream`](#fmodcreatestream) to avoid high memory usage.

<ParamField path="path" type="string" required>
  Resource-relative path to the audio file (e.g. `"sounds/click.wav"`).
</ParamField>

<ParamField path="is3D" type="bool" default="false">
  When `true`, enables 3D spatialization so the sound can be positioned in world space.
</ParamField>

<ParamField path="loop" type="bool" default="false">
  When `true`, the sound loops continuously until explicitly stopped.
</ParamField>

<ResponseField name="soundId" type="integer | false">
  An integer sound ID on success, or `false` on failure.
</ResponseField>

```lua
local soundId = fmodCreateSound("sounds/click.wav", false, false)
if not soundId then
  local code, msg = fmodGetLastError()
  outputDebugString("[FMOD] " .. msg)
end
```

## fmodCreateStream

```lua
fmodCreateStream(path, [is3D=false], [loop=false])
```

Stream a long audio file from disk. Audio is decoded on-the-fly rather than loaded entirely into RAM, making this the preferred choice for music tracks and ambient loops. The trade-off is a slightly higher CPU cost compared to `fmodCreateSound`.

<ParamField path="path" type="string" required>
  Resource-relative path to the audio file (e.g. `"audio/ambient.ogg"`).
</ParamField>

<ParamField path="is3D" type="bool" default="false">
  When `true`, enables 3D spatialization.
</ParamField>

<ParamField path="loop" type="bool" default="false">
  When `true`, the stream loops continuously until explicitly stopped.
</ParamField>

<ResponseField name="soundId" type="integer | false">
  An integer sound ID on success, or `false` on failure.
</ResponseField>

```lua
local musicId = fmodCreateStream("audio/ambient.ogg", false, true)  -- looped stream
local ch = fmodPlaySound(musicId)
fmodSetChannelCategory(ch, "music")
```

## fmodCreateSoundFromMemory

```lua
fmodCreateSoundFromMemory(data, [is3D=false], [loop=false])
```

Load a sound from a raw audio data string — for example, a TEA-decrypted `.wavlocked` buffer read from a custom file format. FMOD copies the buffer internally, so the Lua string can be garbage-collected safely after the call returns.

<ParamField path="data" type="string" required>
  Raw audio data as a Lua string. This is typically the binary contents of an audio file read via `fileRead`.
</ParamField>

<ParamField path="is3D" type="bool" default="false">
  When `true`, enables 3D spatialization.
</ParamField>

<ParamField path="loop" type="bool" default="false">
  When `true`, the sound loops continuously until explicitly stopped.
</ParamField>

<ResponseField name="soundId" type="integer | false">
  An integer sound ID on success, or `false` on failure.
</ResponseField>

```lua
local file = fileOpen("sounds/engine.wavlocked")
local data = fileRead(file, fileGetSize(file))
fileClose(file)
local soundId = fmodCreateSoundFromMemory(data, true, true)
```

## fmodFreeSound

```lua
fmodFreeSound(soundId)
```

Release a sound asset and free its associated audio memory. Any channels currently playing this sound are stopped and invalidated immediately. Do not use the `soundId` after calling this function.

<ParamField path="soundId" type="integer" required>
  The sound ID returned by `fmodCreateSound`, `fmodCreateStream`, or `fmodCreateSoundFromMemory`.
</ParamField>

<ResponseField name="result" type="bool">
  `true` on success, `false` on failure. Call `fmodGetLastError()` for details.
</ResponseField>

```lua
local ok = fmodFreeSound(soundId)
if not ok then
  local code, msg = fmodGetLastError()
  outputDebugString("[FMOD] fmodFreeSound failed: " .. msg)
end
```

<Note>
  Sounds are automatically freed when the resource that created them stops. You do not need to call `fmodFreeSound` in an `onResourceStop` handler unless you want to free memory earlier — for example, after a one-shot SFX that will not be played again during the resource's lifetime.
</Note>

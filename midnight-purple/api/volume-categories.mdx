Volume categories let you control groups of sounds independently. Every channel belongs to `sfx` by default; assign it to `ambient` or `music` for independent volume scaling. This lets players (or server scripts) tune music, sound effects, and ambient audio separately without touching individual channels.

---

## Channel Categories

### `fmodSetChannelCategory(channelId, category)`

Move a live channel to a volume category. Once assigned, the channel's audible volume is scaled by both the category volume and the master volume.

**Parameters**

| Name        | Type             | Description                                                                         |
| ----------- | ---------------- | ----------------------------------------------------------------------------------- |
| `channelId` | integer          | The channel handle returned by `fmodPlaySound`.                                     |
| `category`  | string or integer | `"sfx"` / `0`, `"ambient"` / `1`, or `"music"` / `2`. Default category is `"sfx"`. |

**Returns** `bool` — `true` on success, `false` on failure.

```lua
local musicCh = fmodPlaySound(bgMusicId)
fmodSetChannelCategory(musicCh, "music")
```

---

### `fmodSetCategoryVolume(category, volume)`

Scale all channels in a category simultaneously. Changing the category volume is non-destructive — per-channel volumes set by `fmodSetChannelVolume` are preserved and multiplied on top of the category scalar.

**Parameters**

| Name       | Type             | Description                                                     |
| ---------- | ---------------- | --------------------------------------------------------------- |
| `category` | string or integer | `"sfx"` / `0`, `"ambient"` / `1`, or `"music"` / `2`.         |
| `volume`   | number           | Volume scalar, 0.0 (silent) to 1.0 (full category volume).     |

**Returns** `bool` — `true` on success, `false` on failure.

```lua
fmodSetCategoryVolume("music", 0.5)   -- music at 50%
fmodSetCategoryVolume("sfx", 0.9)     -- SFX at 90%
fmodSetCategoryVolume("ambient", 0.7) -- ambient at 70%
```

---

### `fmodGetCategoryVolume(category)`

Read back the current volume scalar for a category.

**Parameters**

| Name       | Type             | Description                                             |
| ---------- | ---------------- | ------------------------------------------------------- |
| `category` | string or integer | `"sfx"` / `0`, `"ambient"` / `1`, or `"music"` / `2`. |

**Returns** `number` — current volume scalar for the category.

## Master Volume

### `fmodSetMasterVolume(volume)`

Scale all FMOD sounds together by adjusting the FMOD master `ChannelGroup`. This is independent of MTA's native BASS audio pipeline — it will not affect any non-FMOD sounds playing in the server.

**Parameters**

| Name     | Type   | Description                                              |
| -------- | ------ | -------------------------------------------------------- |
| `volume` | number | Volume scalar, 0.0 (silent) to 1.0 (full). Default: `0.5`. |

**Returns** `bool` — `true` on success, `false` on failure.

```lua
fmodSetMasterVolume(0.8)
```

---

### `fmodGetMasterVolume()`

Read back the current FMOD master volume scalar.

**Returns** `number` — current master volume scalar.

## Diagnostics

### `fmodGetVersion()`

Return the FMOD Core library version string linked at runtime. Useful for logging or for gating features that require a minimum FMOD version.

**Returns** `string` — FMOD Core library version (e.g. `"2.02.21"`).

```lua
outputDebugString("FMOD version: " .. fmodGetVersion())
```

---

### `fmodGetLastError()`

Return the last FMOD result code and its human-readable description. Call this immediately after any `fmod*` function returns `false` to understand what went wrong.

**Returns**

| Position | Type    | Description                               |
| -------- | ------- | ----------------------------------------- |
| 1        | integer | FMOD result code (`FMOD_RESULT` enum value). |
| 2        | string  | Human-readable description of the error.  |

```lua
local soundId = fmodCreateSound("missing.wav")
if not soundId then
  local code, msg = fmodGetLastError()
  outputDebugString(string.format("[FMOD] Error %d: %s", code, msg))
end
```

## Named Float Parameters

### `fmodSetParameter(name, value)`

Store a named float value for use by the ambience scripting layer. Parameters are global and can be read back by any resource running on the same client, making them suitable for cross-resource state sharing (e.g. exposing engine RPM to an ambient sound resource).

**Parameters**

| Name    | Type   | Description                  |
| ------- | ------ | ---------------------------- |
| `name`  | string | Parameter identifier string. |
| `value` | number | Float value to store.        |

**Returns** `bool` — `true` on success, `false` on failure.

```lua
fmodSetParameter("engineRPM", 4500.0)
```

---

### `fmodGetParameter(name [, default])`

Read back a named float parameter. If the parameter has not been set yet, `default` is returned instead of raising an error.

**Parameters**

| Name      | Type   | Default | Description                                             |
| --------- | ------ | ------- | ------------------------------------------------------- |
| `name`    | string | —       | Parameter identifier string.                            |
| `default` | number | `0`     | Fallback value returned when the parameter is not set.  |

**Returns** `number` — the stored value, or `default` if the parameter has not been set.

```lua
local rpm = fmodGetParameter("engineRPM", 800.0)
```

## Volume Control Pattern

The snippet below shows a practical settings-menu integration that applies all four volume knobs at once, with safe fallbacks if the user has not saved preferences yet.

```lua
-- Apply user audio settings
fmodSetMasterVolume(getUserSetting("fmod_master") or 0.8)
fmodSetCategoryVolume("music",   getUserSetting("music_vol")   or 0.6)
fmodSetCategoryVolume("sfx",     getUserSetting("sfx_vol")     or 0.9)
fmodSetCategoryVolume("ambient", getUserSetting("ambient_vol") or 0.7)
```
